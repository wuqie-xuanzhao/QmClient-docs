## 三、聊天框鼠标交互现状

当前 `CChat` 已经不再是“纯键盘输入组件”，而是具备了明确的鼠标交互基础设施。

### 3.1 代码证据

`chat.h` 中已经存在：

- `STranslateButtonState m_TranslateButton`
- `bool OnCursorMove(float x, float y, IInput::ECursorType CursorType) override`
- `vec2 GetChatMousePos() const`
- `void RenderTranslateButton(const CUIRect &InputRect)`

`chat.cpp` 中已经存在：

- 翻译按钮区域命中检测
- 鼠标按下/抬起状态处理
- 翻译按钮渲染
- 光标位置换算与聊天坐标系映射
- 语言菜单弹出与切换

### 3.2 现有交互能力

当前聊天框鼠标交互至少包含：

- 输入框区域的鼠标参与
- 翻译按钮悬停检测
- 翻译按钮点击处理
- 翻译相关弹出菜单
- 光标渲染与坐标转换

所以旧文档中“聊天框完全不支持鼠标交互”的结论已经失效。

## 四、翻译按钮现状

### 4.1 输入框旁翻译按钮

当前实现是在聊天输入框旁渲染一个翻译按钮，而不是早期方案里设想的“给每条历史聊天消息都加一个独立按钮”。

相关代码：

- `chat.cpp:2078` 附近计算按钮尺寸与输入框可用宽度
- `chat.cpp:2171` 附近创建按钮矩形并调用 `RenderTranslateButton`
- `chat.cpp:2510` 之后实现 `RenderTranslateButton`

### 4.2 当前按钮职责

该按钮当前主要承担的是“出站翻译控制中心”的角色：

- 显示当前出站翻译开关状态
- 允许点击切换自动出站翻译
- 打开语言/后端相关的小型菜单
- 配合目标语言选择完成发送前翻译体验

这和旧稿中“鼠标悬停某条聊天消息，再点击该消息行尾按钮进行入站翻译”的设计目标并不完全相同。  
也就是说：**按钮已实现，但最终落地形态和早期方案有差异。**

## 五、出站翻译现状

### 5.1 功能链路

当前代码里已经有完整的出站翻译链路：

- `CTranslate::ShouldAutoTranslateOutgoing`
- `CTranslate::StartAutoOutgoingTranslate`
- `CTranslate::TryTranslateOutgoingChat`
- `CChat::SendChatQueued(int Team, const char *pLine, bool AllowOutgoingTranslation)`

### 5.2 触发方式

当前出站翻译至少支持两种路径：

1. 自动出站翻译  
   由 `qm_translate_auto_outgoing` 控制，发送前自动翻译

2. 目标语言后缀翻译  
   解析消息中的目标语言标记后再发出翻译结果

因此旧文档中“发送逻辑尚未完成”或“自动出站翻译未实现”的表述，也已经过时。

## 六、当前配置项

与聊天框翻译按钮和出站翻译直接相关的配置，当前至少包括：

| 配置项 | 说明 |
|--------|------|
| `qm_translate_auto_outgoing` | 自动翻译发送消息 |
| `qm_translate_auto_outgoing_mode` | 自动出站翻译触发模式 |
| `qm_translate_outgoing_target` | 出站翻译目标语言 |
| 多组 `qm_translate_*_color` | 翻译按钮及 UI 颜色相关配置 |

旧文档中假设的 `qm_translate_show_button` 并不是当前实现的核心配置项，至少从当前代码核对结果看，不应再把它写成主要开关。
